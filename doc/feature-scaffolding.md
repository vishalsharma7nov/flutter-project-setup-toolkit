# Feature scaffolding

Create new feature modules with consistent folder structure via CLI or Feature Studio.

## CLI

```bash
dart run :make_feature --project /path/to/app
dart run :make_feature --project . --feature ride_history
dart run :make_feature --project . --feature settings --base-path lib/modules
dart run :make_feature --project . --feature auth --dry-run
dart run :make_feature --project . --feature auth --state-management bloc
```

From app `scripts/` (generated by setup):

```bash
./scripts/make-feature.sh ride_history
./scripts/make-feature.sh settings ./lib/modules
```

### Flags

| Flag | Description |
|------|-------------|
| `--project` / `-p` | Flutter project root |
| `--feature` / `-f` | Feature name (prompted if omitted) |
| `--base-path` | Override `architecture.feature_base_path` from config |
| `--state-management` | `bloc`, `riverpod`, `provider`, `getx`, `none` |
| `--dry-run` | Print planned paths without writing |
| `--yes` | Skip prompts (requires `--feature`) |
| `--gui` / `--view feature` | Open Feature Studio in Toolkit Studio |

### Naming

- Input `my-feature` → folder `my_feature`, file prefix `my_feature_`
- Invalid Dart identifiers are sanitized automatically

## Layout by preset

Structure depends on `architecture.preset` in config. Examples:

**`feature_first_clean`** (default):

```text
lib/features/ride_history/
├── data/
│   ├── datasources/
│   └── repositories/
├── domain/
│   ├── entities/
│   ├── repositories/
│   └── usecases/
└── presentation/
    ├── pages/
    └── widgets/
```

With `state_management: bloc`, also adds `presentation/bloc/`.

**`getx_module`**: `lib/modules/<feature>/` with GetX-oriented folders.

**`micro_feature`**: `packages/<feature>/lib/src/...`

**`custom`**: paths from your JSON template.

See [architecture.md](architecture.md) for the full preset catalog.

## Test mirror

When `architecture.bootstrap.scaffold_test_mirror` is `true`, each scaffolded `.dart` file gets a matching stub under `test/`:

```text
test/lib/features/ride_history/presentation/pages/ride_history_page_test.dart
```

## Feature Studio

Open from the hub or directly:

```bash
dart run :toolkit_studio --view feature --project .
```

Workflow:

1. Load project — reads `release-toolkit.config.json`
2. Enter feature name and options
3. Preview tree
4. Apply or save config changes (preset, API protocol, external SDK)

## During setup

Setup wizard can scaffold the first feature:

```bash
dart run :setup_project --project . --yes --make-feature authentication
```

Or enable **Scaffold a feature to start with** in Setup Studio step 4.

## Safe behavior

- Never overwrites existing files — skips paths that already exist
- Only creates directories and empty `.dart` files (unless starter code is enabled in future)
- Does not modify unrelated project files
